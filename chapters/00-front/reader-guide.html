Reader’s Guide

What this book assumes

The book assumes a working familiarity with vectors and matrices. A reader should be comfortable with matrix multiplication, transposes, inverses when they exist, eigenvalues at the level of a first linear algebra course, and the idea that a matrix represents a linear map.

The book also assumes basic single-variable calculus. Derivatives and integrals are used from the beginning. Ordinary differential equations are introduced only in the simple form needed for state space models. The scalar equation

\[ x'(t)=ax(t)+bu(t) \]

is treated before the matrix equation, because it already contains the main roles. The coefficient \(a\) governs the internal evolution of the state, while \(b\) determines how the input enters.

Some deep learning vocabulary is assumed. A reader should know what a neural network layer is, what a learned parameter is, what a sequence of token representations looks like, and why gradients are used to train models. No prior knowledge of state space models, the Structured State Space sequence model (S4), Mamba, control theory, signal processing, or numerical analysis is assumed.

What is introduced in context

Specialised objects are introduced when they become necessary. The matrix exponential appears because it solves the continuous-time state equation. Zero-order hold appears because sampled sequence data does not specify what happens between samples. Toeplitz matrices appear because a time-invariant convolution has the same weights along each diagonal. Generating functions appear because a long kernel can be collapsed into a single matrix inverse, the resolvent.

Specialised terms are introduced where the calculation first needs them, rather than collected up front as conventions from different literatures.

Each chapter ends with a short Notes and references section. Those sources are named so the reader can place the classical results. The derivations in the text do not depend on them.

Mathematical notation

Equations fix the objects under discussion. The notation distinguishes continuous and sampled time. A trajectory written as \(x(t)\) evolves over continuous time, while a sequence written as \(x_k\) is indexed by discrete positions. The notation also distinguishes single-input and multi-input systems. A vector \(B\in\mathbb R^{N\times 1}\) injects one input coordinate into an \(N\)-dimensional state, while a matrix \(B\in\mathbb R^{N\times p}\) injects \(p\) input coordinates.

Notation sections record the conventions used throughout the book. Individual chapters restate local conventions when a symbol changes role or when a formula is easiest to read with temporary notation. Later chapters reuse these objects while changing one assumption at a time.
